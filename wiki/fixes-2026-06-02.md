## 🟢 Легкі (Easy) — Quick Wins

### E1: CRLF → LF нормалізація `story-reader-responsive.tsx`
- **Проблема:** `components/story-reader-responsive.tsx` використовує `\r\n` (Windows line endings)
- **Фікс:** `sed -i 's/\r$//' components/story-reader-responsive.tsx`
- **Пріоритет:** LOW
- **Час:** 1 хвилина

### E2: Міграція `SaveSlot`/`PlaybackState` з barrel import
- **Файл:** `stores/use-app-store.ts:16`
- **Було:** `import type { SaveSlot, PlaybackState } from '@/lib/types';`
- **Стане:** `import type { PlaybackState } from '@/lib/engine/types';` + `import type { SaveSlot } from '@/lib/story-domain';`
- **Пріоритет:** LOW
- **Час:** 5 хвилин

### E3: Додати `colors.backdrop` та `colors.overlay` в тему
- **Проблема:** 6 компонентів використовують hardcoded `rgba()` для overlay/backdrop
- **Фікс:** Додати токени `backdrop: 'rgba(0,0,0,0.45)'` та `overlay: 'rgba(0,0,0,0.3)'` в `constants/theme-colors.json` (з oklch-еквівалентами для світлої/темної теми)
- **Потім замінити:** `ReaderControls.tsx`, `BlockLibraryPanel.tsx`, `MigrationErrorBanner.tsx`, `WebSidebar.tsx`
- **Пріоритет:** MEDIUM
- **Час:** 15 хвилин

### E4: Guarged console.log в `api.ts`
- **Файл:** `lib/_core/api.ts:89,91,98`
- **Фікс:** Обгорнути всі `console.log` у блок `if (__DEV__) { ... }`
- **Примітка:** Вже є `__DEV__`-guarded деякі логи, але не всі
- **Пріоритет:** MEDIUM
- **Час:** 5 хвилин

### E5: Accessibility hints у ReaderControls та ReaderChoices
- **Файли:** `components/reader/ReaderControls.tsx`, `components/reader/ReaderChoices.tsx`
- **Фікс:** Додати `accessibilityHint` до кожної кнопки (наприклад, "Double tap to toggle auto-play")
- **Пріоритет:** LOW
- **Час:** 10 хвилин

### E6: Видалення `file://` з `isSafeUri()` або обмеження native-only
- **Файл:** `lib/story-validator.ts:27`
- **Фікс:** Прибрати `file://` з `allowedPrefixes`, або додати коментар "native-only"
- **Пріоритет:** LOW
- **Час:** 5 хвилин

## 🟡 Середні (Medium) — З залежностями

### M1: Phone layout overflow у SceneComposer
- **Файл:** `components/editor/SceneComposer.tsx:327-353`
- **Проблема:** 5 табів у одному ряду на телефоні переповнюються (< 360px)
- **Фікс:** Обернути таби в горизонтальний `ScrollView` або зменшити кількість видимих табів до 3 + "More" кнопка
- **Пріоритет:** MEDIUM
- **Час:** 30 хвилин

### M2: `moduleUriCache` / `modulePlayableCache` — додати LRU eviction

- **Фікс:** Додати розмір кеша (наприклад, 50 записів) + евікція найстарішого запису
- **Пріоритет:** MEDIUM
- **Час:** 20 хвилин

### M3: Accessibility для BlockLibraryPanel, PropertiesPanel, TimelinePanel

- **Фікс:** Додати `accessibilityRole="button"` + `accessibilityLabel` до кожного інтерактивного елемента
- **Пріоритет:** LOW (editor рідко використовується з screen reader)
- **Час:** 45 хвилин

### M4: Перевірити `isRunningInPreviewIframe()` на dead code

- **Фікс:** `grep -rn "isRunningInPreviewIframe" --include="*.ts" --include="*.tsx" .` — якщо 0 викликів, видалити
- **Пріоритет:** LOW
- **Час:** 10 хвилин

## 🔴 Складні (Hard) — Рефакторинг

### H1: Декомпозиція DocumentSceneEditor (God Component)
- **Файл:** `components/document-editor/DocumentSceneEditor.tsx` (719 LOC, 34 хуки)
- **Ціль:** Розбити на 4-6 підкомпонентів + 3-4 хуки
- **План:**
  1. Винести `scrollToWritingPosition` + `followWriting` → `useDocumentScroll` хук
  2. Винести блокові операції → `useBlockOperations` хук
  3. Винести `lineDrafts` → `useLineDrafts` хук
  4. Винести UI: `DocumentPage`, `DocumentBlockRenderer`, `DocumentLineInput`
  5. Зменшити основний компонент до < 200 LOC
- **Пріоритет:** HIGH
- **Час:** 2-3 години
- **Ризик:** Високий — зміна структури без тестів

### H2: Accessibility для DocumentSceneEditor
- **Файл:** `components/document-editor/DocumentSceneEditor.tsx`
- **Фікс:** Додати `accessibilityLabel` для кожного текстового блоку, `accessibilityRole` для кнопок, `accessibilityHint` для команд
- **Пріоритет:** HIGH
- **Час:** 1 година
- **Залежить від:** H1 (після декомпозиції лше додавати a11y)

### H3: Smart scroll для DocumentSceneEditor
- **Файл:** `components/document-editor/DocumentSceneEditor.tsx:160-184`
- **Проблема:** `followWriting()` завжди прокручує до кінця сторінки
- **Фікс:** Реалізувати `smartFollow(cursorY)`:
  - Тільки прокручувати якщо cursorY виходить за межі viewport
  - Трекати cursor position через `onSelectionChange`
  - Зберігати viewport height у `useRef`
- **Пріоритет:** MEDIUM
- **Час:** 1.5 години
- **Залежить від:** H1 (краще робити в `useDocumentScroll`)
