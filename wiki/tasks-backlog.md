## Архітектурні завдання

### A3: Dependency Injection для сервісів
- **Проблема:** Module-level singletons (AudioPlayerService, AudioLibraryService, AudioTriggerScheduler)
- **Файли:** `lib/audio-manager-enhanced.ts`, `lib/audio-player-service.ts`, `lib/audio-library-service.ts`
- **Рішення:** Впровадити DI контейнер або factory pattern
- **Пріоритет:** Середній
- **Оцінка:** 4-6 годин

### A5: Шар сервісів
- **Проблема:** Бізнес-логіка розкидана по хукам та компонентах
- **Рішення:** Виділити окремий шар сервісів (story-service, audio-service, character-service)
- **Пріоритет:** Середній
- **Оцінка:** 8-12 годин

### A7: Єдина точка входу для аудіо
- **Проблема:** AudioPlayerService, AudioLibraryService, AudioTriggerScheduler — окремі сервіси
- **Рішення:** Об'єднати в AudioFacade з чітким API
- **Пріоритет:** Середній
- **Оцінка:** 4-6 годин

### A8: Модульна архітектура
- **Проблема:** Немає чіткого розділення на модулі
- **Рішення:** Впровадити feature-based структуру (features/story, features/editor, features/reader)
- **Пріоритет:** Низький
- **Оцінка:** 12-16 годин

### H2: Тонкі обгортки story-context-enhanced.ts
- **Проблема:** Всі функції — один рядок делегації до Zustand
- **Рішення:** Прибрати або додати реальну логіку (валідація, трансформація)
- **Пріоритет:** Низький
- **Оцінка:** 2-4 години

## Завдання з продуктивності

### P1: Міграція ефектів на react-native-reanimated
- **Проблема:** Кожен ефект створює десятки-сотні Animated.Value об'єктів
- **Файли:** `components/effects/*.tsx` (SnowEffect, RainEffect, ParticlesEffect, SparklesEffect, StormEffect, FogEffect, SunraysEffect)
- **Рішення:** Мігрувати на react-native-reanimated worklets
- **Пріоритет:** Високий
- **Оцінка:** 16-24 години

### P4: LegoCanvas O(n^2) snap detection
- **Проблема:** Перебір всіх атомів для кожного перетягуваного атома
- **Файл:** `components/lego-editor/LegoCanvas.tsx`
- **Рішення:** Впровадити spatial hashing або quadtree для O(n) або O(n log n)
- **Пріоритет:** Середній
- **Оцінка:** 6-8 годин

### P5: character-library.ts — Лінійний пошук
- **Проблема:** Всі операції використовують лінійний пошук
- **Файл:** `lib/character-library.ts`
- **Рішення:** Використовувати Map<string, Character> для O(1) lookup
- **Пріоритет:** Низький
- **Оцінка:** 2-3 години

### P6: Віртуалізація довгих списків
- **Проблема:** Довгі списки рендеряться повністю
- **Рішення:** Впровадити FlatList з windowing для списків блоків, персонажів, тощо
- **Пріоритет:** Середній
- **Оцінка:** 4-6 годин

### P7: Мемоізація компонентів
- **Проблема:** Багато компонентів не використовують React.memo, useMemo, useCallback
- **Рішення:** Проаналізувати та додати мемоізацію в критичних місцях
- **Пріоритет:** Середній
- **Оцінка:** 6-8 годин

### P8: Lazy loading компонентів
- **Проблема:** Всі компоненти завантажуються одразу
- **Рішення:** Впровадити React.lazy() + Suspense для екранів та важких компонентів
- **Пріоритет:** Низький
- **Оцінка:** 3-4 години
