# Dev Server Fix - 2026-05-06

## Проблема
Dev-сервер (Metro + Expo) не запускається. Команда `pnpm dev` або `pnpm dev:metro` зависає без виводу, порт 8081 не слухається.

Початкова помилка: `Error loading Metro config` / `Error reading Expo config`

## Що було зроблено

### 1. Виправлення app.config.ts
- Видалено імпорт `scripts/load-env.js` (використовував ESM імпорт, який Expo не міг обробити)
- Спроба №1: Додано `require('fs')` та `require('path')` — **не спрацювало**, оскільки проект використовує ESM (`"type": "module"` у package.json)
- Спроба №2: Використано `dotenv` (вже встановлений): `import { config as dotenvConfig } from 'dotenv'`
- Додано `// @ts-nocheck` для вимкнення перевірки типів (конфлікти в node_modules)
- Перейменовано імпорт dotenv, щоб уникнути конфлікту зі змінною `config`

### 2. Створення app.config.js
- Створено CommonJS версію конфігурації (`app.config.js`) для тестування
- Перейменовано `app.config.ts` → `app.config.ts.bak`, щоб Expo використовував `.js` версію
- **Результат**: Metro все одно не запустився

### 3. Додаткові спроби
- Встановлено `@babel/parser` та `@babel/types` (усунуто warnings у лінтері)
- Спроба запуску з `--clear` прапорцем
- Перевірка портів 3000 та 8081 — вільні
- Перевірка `expo --help` — працює коректно

## Статус
❌ **Проблема не вирішена**. Metro не запускається, немає жодного виводу в лог (ані stdout, ані stderr). Процес просто зависає.
