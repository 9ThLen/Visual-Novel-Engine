## Знайдені баги

### 1. `components/story-reader.tsx` — неіснуюча властивість персонажа + відсутність null check

**Проблема:**
- `char.imageUri` — `CharacterSprite` має `uri`, не `imageUri`. Зображення персонажів ніколи не відображались.
- `scene.text.split('\n\n')` — crash при `null/undefined` тексті.

**Виправлення:**
- `char.imageUri` → `char.uri` з перевіркою `char.uri ? { uri: char.uri } : undefined`
- `scene.text.split('\n\n')` → `(scene.text || '').split('\n\n')`

**Файл:** `components/story-reader.tsx:31,75`

---

### 2. `components/story-reader-responsive.tsx` — неіснуюча властивість персонажа

**Проблема:**
- `char.imageUri` — те саме, що в `story-reader.tsx`. Зображення не резолвились.

**Виправлення:**
- `char.imageUri` → `char.uri`

**Файл:** `components/story-reader-responsive.tsx:130,133`

---

### 3. `lib/story-validator.ts` — валідатор перетворював персонажів на рядки

**Проблема:**
```typescript
// До: персонажі ставали рядками "[object Object]"
characters: characters.map((c: any) => this.sanitizeString(String(c))),
```
`String(c)` для об'єкта повертає `"[object Object]"`. Тип `StoryScene.characters` — `CharacterSprite[]`, але валідатор повертав `string[]`.

**Виправлення:**
- Додано імпорт `CharacterSprite` з `./character-types`
- Валідатор тепер приймає:
  - **String** (старий формат): створює `CharacterSprite` з `id=name=рядок, uri=''`
  - **Object**: валідує `id`, `name`, `uri` (або `imageUri` для сумісності), `tags`, `position`, `scale`, `expression`

```typescript
// Після
characters: characters.map((c: unknown) => {
  if (typeof c === 'string') {
    return { id: c, name: c, uri: '', createdAt: Date.now() } as CharacterSprite;
  }
  // ... валідація CharacterSprite
}),
```

**Файл:** `lib/story-validator.ts:126-142`

### 4. `lib/audio-library.ts` — використання generic Error

- `throw new Error('Audio item not found')` — замість наявної системи `ErrorHandler`

**Виправлення:**

**Файл:** `lib/audio-library.ts:76`
