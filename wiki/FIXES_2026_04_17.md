### 4.1 Cleanup функции в useEffect
**Проблема:** 40+ `useEffect` без cleanup функций, риск утечек памяти

**Исправление в app/reader.tsx:**
```tsx
useEffect(() => {
  let isMounted = true;
  
  // Async operations
  resolveAssetUri(currentScene.musicUri).then((uri) => {
    if (isMounted && uri) {
      audioManager.crossFade('bgm', uri, settings.bgmVolume);
    }
  }).catch(() => {
    // Handle error
  });

  return () => {
    isMounted = false; // Cleanup
  };

```

## 8. Исправление загрузки аудио

### 8.1 Бесконечный цикл аудио
**Проблема:** Audio manager повторял попытки загрузки невалидных файлов, вызывая бесконечные re-render циклы

**Исправления:**

**В app/reader.tsx:**
```tsx
useEffect(() => {
  // Проверка валидности URI перед загрузкой
  if (currentScene.musicUri && currentScene.musicUri.trim()) {
    resolveAssetUri(currentScene.musicUri).then((uri) => {
      if (isMounted && uri) {
        audioManager.crossFade('bgm', uri, settings.bgmVolume);
      } else if (isMounted) {
        audioManager.stop('bgm');
      }
    }).catch(() => {
      if (isMounted) audioManager.stop('bgm');
    });
  } else {
    // Остановить аудио если нет URI
    audioManager.stop('bgm');
  }
}, [currentScene?.id]);
```

**В lib/audio-manager.ts:**
```tsx
// Удалены спамные консоль логи при ошибках (silent fail)
try {
  const player = new AudioModule.AudioPlayer(uri, 100, false);
  // ... setup
} catch (err) {
  // Silently fail to avoid spam
}
```

### 8.2 Результат
- Нет больше циклических ошибок аудио
- Консоль чистая от ошибок загрузки некорректных файлов
- Приложение не зависает из-за аудио

## 12. Рекомендации

### Для добавления аудио
1. Найти или создать аудиофайлы в `assets/sounds/`
2. Обновить пути в `demo-story.json` или `demo-story-advanced.json`
3. Добавить маппинг в `lib/asset-resolver.ts`

### Для добавления новых изображений
1. Поместить изображения в `assets/background/` или `assets/charakters/`
2. Добавить маппинг в `BUNDLED_ASSETS` в `lib/asset-resolver.ts`
3. Использовать в stories с полным путём: `assets/background/filename.png`

### Для просмотра логов
- На телефоне: используй Expo Go app логи
- На web: откройте браузерную консоль (F12)
- Ищите `[StoryReader]` сообщения для отладки активов
