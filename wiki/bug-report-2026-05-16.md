## 2. Критичні баги (нові)

### K1: `lib/story-validator.ts` — Неповна валідація блоків
- **Рядок:** 128
- **Проблема:** `blocks: data.blocks, // TODO: Add validation for blocks`
- **Вплив:** Дані блоків не валідуються при імпорті історії. Шкідливі або некоректні дані можуть пройти без перевірки.
- **Рівень:** Середній (не критичний, бо блоки — складна система)

### K2: `lib/story-repository.ts` — Race condition при збереженні
- **Рядки:** 43-66, 68-82
- **Проблема:** `saveStory` та `deleteStory` викликають `getAllStoriesMetadata()` для читання, потім пишуть назад. Якщо два виклики відбуваються одночасно, один перезапише інший (read-modify-write race).
- **Вплив:** Втрата даних при паралельних операціях.
- **Рівень:** Високий

### K3: `lib/error-handler.ts` — Безкінечна рекурсія при помилці в listener
- **Рядки:** 108-113
- **Проблема:** Якщо listener кидає помилку, `ErrorHandler.handle()` викликає сам себе з тим самим шаблоном, що може призвести до рекурсії.
- **Вплив:** Stack overflow при помилці в error listener.
- **Рівень:** Середній (try/catch частково захищає)

### K4: `lib/audio-player-service.ts` — Витік при виклику stop з fadeOut
- **Рядки:** 75-92
- **Проблема:** При `stop(trackId, fadeOut)` створюється Promise, який ніколи не резолвиться, якщо `_fade` викликає `onDone` після видалення track з `this.tracks`. Якщо track видаляється до завершення fade, Promise висітиме назавжди.
- **Вплив:** Memory leak при частих stop/start з fadeOut.
- **Рівень:** Високий

### K5: `lib/audio-player-service.ts` — Неконсистентний стан при crossFade
- **Рядки:** 107-125
- **Проблема:** `crossFade` викликає `_fade` для старого track (async) та `play` для нового (sync). Між цим стани можуть бути неконсистентні — обидва tracks грають одночасно без синхронізації.
- **Вплив:** Артефакти звуку при кроссфейді.
- **Рівень:** Середній

### K6: `components/effects/StormEffect.tsx` — Неконтрольована анімація
- **Рядки:** 20-65
- **Проблема:** `triggerFlash` викликається через `setInterval`, але немає гарантії що компонент ще змонтований коли анімація завершиться. `isFlashingRef` не скидається при unmount.
- **Вплив:** Потенційний memory leak та оновлення стану демонтованого компонента.
- **Рівень:** Середній

### K7: `hooks/useAutoSave.ts` — Витік таймера
- **Рядки:** 22-49
- **Проблема:** `useEffect` залежить від `playbackState` та `currentStory` (об'єкти). При кожній зміні створюється новий `setTimeout`, але старий не очищується якщо `useEffect` не встиг запустити cleanup.
- **Вплив:** Накопичення таймерів при частих змінах стану.
- **Рівень:** Низький

### K8: `hooks/useReaderInitialization.ts` — Race condition
- **Рядки:** 24-73
- **Проблема:** `initializeReader` використовує `requestId` для скасування, але `await setCurrentStory()` може призвести до того що старий запись перезапише новий.
- **Вплив:** Неправильний стан при швидкій навігації між історіями.
- **Рівень:** Високий

## 3. Високі проблеми (нові)

### H1: `lib/story-context.tsx` — Подвійний стейт-менеджмент
- **Рядки:** 124-254
- **Проблема:** React Context обгортає Zustand. Кожен компонент, що використовує `useStoryState()`, перерендерюється при будь-якій зміні в Zustand store.
- **Вплив:** Нестабільна продуктивність, подвійні re-renders.
- **Рекомендація:** Прибрати Context, використовувати Zustand напряму.

### H2: `lib/story-context-enhanced.ts` — Тонкі обгортки без логіки
- **Рядки:** 1-59
- **Проблема:** Кожна функція — один рядок, що делегує до `useAppStore.getState()`. Цей шар не додає логіки.
- **Вплив:** Непотрібна індирекція, ускладнює рефакторинг.
- **Рекомендація:** Прибрати або додати реальну логіку (валідація, трансформація).

### H3: `lib/block-tree.ts` — JSON.parse(JSON.stringify()) для клонування
- **Рядок:** 114
- **Проблема:** `duplicateBlock` використовує JSON roundtrip для deep clone. Втрачає undefined, Date, функції.
- **Вплив:** Потенційна втрата даних при дублюванні блоків.
- **Рекомендація:** Використовувати структурне клонування.

### H4: `lib/block-tree.ts` — Непотрібний ROOT_BLOCK імпорт
- **Рядок:** 3
- **Проблема:** `import { ROOT_BLOCK } from './block-types'` — ROOT_BLOCK не використовується у файлі.
- **Вплив:** Мертвий імпорт.

### H5: `lib/scene-persistence.ts` — Дублювання логіки автозбереження
- **Рядки:** 16-72
- **Проблема:** `useAutoSave` хук дублює логіку автозбереження, яка вже є в `useAutoSave.ts`.
- **Вплив:** Два механізми автозбереження можуть конфліктувати.

### H6: `lib/asset-resolver.ts` — Кеш без обмеження
- **Рядок:** 9
- **Проблема:** `uriCache` — Map без обмеження розміру. При довгій роботі може накопичувати записи.
- **Вплив:** Memory leak при довгій роботі.
- **Рекомендація:** Використовити LRU cache з обмеженням.

### H7: `lib/asset-resolver.ts` — data: URI пропускає валідацію
- **Рядок:** 103
- **Проблема:** `if (uri.startsWith('blob:') || uri.startsWith('data:')) return uri;` — data URI повертається без перевіри на небезпечні протоколи.
- **Вплив:** Можливе XSS через `data:text/javascript`.
- **Рівень:** Високий (безпека)

### H8: `lib/inventory-context.tsx` — Нестабільний hasItem/hasItems
- **Рядки:** 24-25
- **Проблема:** `hasItem` та `hasItems` створюються в `useMemo`, але залежать від `inventory.items`. При зміні items всі компоненти, що використовують ці функції, перерендерюються.
- **Вплив:** Нестабільна продуктивність.

### H9: `lib/i18n-context.tsx` — Імпорт після оголошення
- **Рядок:** 34
- **Проблема:** `import translations from './translations.json';` розміщений після оголошення констант та інтерфейсів.
- **Вплив:** Порушення конвенцій, потенційні проблеми з порядком ініціалізації.

### H10: `components/effects/RainEffect.tsx` — Неконтрольований рекурсивний таймер
- **Рядки:** 66-100
- **Проблема:** `runIteration` викликає сам себе рекурсивно через `.start(() => runIteration())`. При швидкій зміні `drops` старі ітерації не скасовуються.

### H11: `components/effects/ParticlesEffect.tsx` — Аналогічна проблема
- **Рядки:** 65-127
- **Проблема:** Аналогічна RainEffect — рекурсивний таймер без повного скасування.

### H12: `components/effects/SnowEffect.tsx` — Аналогічна проблема

- **Проблема:** Аналогічна — створення Animated.Value у useEffect без очищення при зміні параметрів.

## 7. Проблеми безпеки

### S1: XSS через data: URI
- **Файл:** lib/asset-resolver.ts, рядок 103
- **Проблема:** `data:` URI пропускається без валідації.
- **Вплив:** Можливе виконання JavaScript.

### S2: Неповна санитизація XSS
- **Файл:** lib/story-validator.ts
- **Проблема:** sanitizeText не обробляє всі вектори XSS.
- **Вплив:** Можливе виконання JavaScript.

### S3: file:// протокол
- **Файл:** lib/story-validator.ts, рядок 237
- **Проблема:** file:// дозволений.
- **Вплив:** Читання локальних файлів.

### S4: wildcard postMessage
- **Файл:** lib/_core/manus-runtime.ts
- **Проблема:** postMessage з wildcard origin.
- **Вплив:** Витік даних.

### S5: Відсутність CSRF захисту
- **Проблема:** API запити не мають CSRF токенів.
- **Вплив:** CSRF атаки.

### S6: Відсутність rate limiting
- **Проблема:** Немає обмеження кількості запитів.
- **Вплив:** DoS атаки.

### S7: Відсутність input sanitization для імен файлів
- **Файл:** lib/asset-resolver.ts
- **Проблема:** Імена файлів не очищаються від спеціальних символів.
- **Вплив:** Path traversal.

### S8: Відсутність валідації розміру файлів
- **Проблема:** Немає обмеження розміру імпортованих файлів.
- **Вплив:** DoS через великі файли.

### A1: Три паралельні системи редагування
- **Проблема:** Lego, Block, Node — три незалежні системи.
- **Вплив:** Потроєння навантаження на підтримку.

- **Проблема:** React Context + Zustand.

### A3: Відсутність Dependency Injection
- **Проблема:** Module-level singletons.
- **Вплив:** Складність тестування.

- **Проблема:** 10+ модулів імпортують useAppStore напряму.

### A5: Відсутність шару сервісів
- **Проблема:** Бізнес-логіка розкидана по хукам та компонентах.

- **Проблема:** StoryRepository дублює логіку Zustand.

### A7: Відсутність єдиної точки входу для аудіо
- **Проблема:** AudioPlayerService, AudioLibraryService, AudioTriggerScheduler — окремі сервіси.
- **Вплив:** Складність координації.

### A8: Відсутність модульної архітектури
- **Проблема:** Немає чіткого розділення на модулі.
- **Вплив:** Складність навігації по коду.

### A9: Відсутність документації архітектури
- **Проблема:** Архітектурні рішення не задокументовані.
- **Вплив:** Складність онбордингу.

### A10: Відсутність версіонування API
- **Проблема:** API не має версій.
- **Вплив:** Складність міграції.
