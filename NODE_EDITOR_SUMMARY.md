## Ключевые возможности

### ✅ Визуальный граф
- Все сцены отображаются как узлы
- Связи между сценами видны как стрелки
- Иерархическая раскладка (BFS)

### ✅ Интерактивный canvas
- Zoom: 50% - 200%
- Pan: перетаскивание фона
- Grid background для ориентации

### ✅ Профессиональные узлы
- ID сцены
- Превью текста (60 символов)
- Индикаторы медиа: 🖼 🎵 🗣
- Счетчик выборов
- Система предупреждений ⚠️
- Badges: START, END

### ✅ Создание связей
- Tap на + handle источника
- Tap на целевой узел
- Автоматическое создание choice

### ✅ Панель редактирования
- Текст диалога
- Загрузка изображений
- Загрузка аудио (голос, музыка)
- Управление выборами
- Индикатор изменений
- Кнопка сохранения

### ✅ Валидация
- Нет текста диалога
- Тупиковые сцены
- Сломанные связи

### Data Flow

```
Story (AsyncStorage)
  ↓
StoryContext
  ↓
NodeEditor
  ↓
├─ NodeCanvas (visualization)
│   ├─ StoryNode × N
│   └─ SVG Edges
└─ SceneEditorPanel (editing)
    ↓
storyContextEnhanced (CRUD)
    ↓

```

## Использование

### Открыть редактор

1. Главный экран → Editor
2. Выбрать историю
3. Нажать **🗺 Graph**

### Редактировать сцену

1. Кликнуть на узел
2. Редактировать в правой панели
3. Нажать **💾 Save**

### Создать связь

1. Нажать **+** на узле-источнике
2. Кликнуть на целевой узел
3. Связь создана автоматически

### Удалить сцену

1. Долгое нажатие на узел
2. Выбрать "Delete"
3. Подтвердить

## Технические детали

### Производительность

- **Memoization**: Layout вычисляется только при изменении истории
- **Lazy rendering**: Только видимые узлы (будущее улучшение)
- **Batched SVG**: Все edges в одном SVG
- **Optimized events**: Debounced pan/zoom

### Типы

```typescript
interface NodeData {
  id: string;
  position: { x: number; y: number };
  size: { width: number; height: number };
  isStart: boolean;
  isEnd: boolean;
  hasImage: boolean;
  hasAudio: boolean;
  hasVoice: boolean;
  choiceCount: number;
  textPreview: string;
  warnings: string[];
}

interface EdgeData {
  id: string;
  sourceNodeId: string;
  targetNodeId: string;
  label: string;
  choiceId: string;
}
```

### Алгоритм раскладки

**Hierarchical BFS:**
1. BFS от startSceneId
2. Присвоить уровень каждой сцене
3. Группировать по уровням
4. Позиционировать горизонтально внутри уровня
5. Недостижимые сцены → в конец

- ✅ Split-panel layout
- ✅ Профессиональный UX
- ✅ Масштабируется для больших историй
