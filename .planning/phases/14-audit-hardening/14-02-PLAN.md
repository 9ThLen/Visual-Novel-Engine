    **1. Add `initPromise` singleton guard to `lib/audio-player-service.ts`:**

    In the `AudioPlayerService` class (around L24), add:
    ```ts
    /** Module-level singleton initialization promise (runs at most once). */
    private initPromise: Promise<void> | null = null;
    ```

    Replace the `initialize()` method (L33-41):
    ```ts
    async initialize(): Promise<void> {
      if (this.initPromise) return this.initPromise;
      this.initPromise = (async () => {
        if (this.initialized) return;
        try {
          await setAudioModeAsync({ playsInSilentMode: true });
          this.initialized = true;
        } catch (err) {
          // Reset on failure so a future call can retry
          this.initPromise = null;
          ErrorHandler.handle('AudioPlayer init failed', err, ErrorCategory.MEDIA, ErrorSeverity.LOW);
        }
      })();
      return this.initPromise;
    }
    ```

    The `initialized` flag is preserved (still tracks completion). The new `initPromise` ensures concurrent `initialize()` calls share the same promise (avoids race where two callers each start an `setAudioModeAsync` call).
