<objective>
Add one shared, pure migration module that converts legacy
`action`/`fadeDuration`/`pitchVariation`-shaped Music/Sound block data into the
new `mode`/`boundTo`/`fadeIn`/`fadeOut` shape from 18-00, and wire it into
every canonical scene-load boundary (persisted app state, bundled stories,
story import validation) so old saved/imported stories never reach the reader
or preview runtime in the legacy shape.
</objective>

<tasks>

<task type="auto">
  <name>Create the shared audio block migration module</name>
  <files>lib/audio-block-migration.ts</files>
  <action>
    New file, modeled on `lib/character-migration.ts`'s pattern (pure
    functions, no side effects, safe to call repeatedly/idempotent). Export:
    - `migrateMusicBlockData(data: unknown): MusicBlockData` — if `data`
      already has `mode`, return it unchanged (already migrated). Otherwise
      treat it as the legacy shape and map per 18-CONTEXT.md D-08:
      `action:'play'` -> `{ mode:'track', boundTo:'continuous', fadeIn:
      (fadeDuration ?? 1000)/1000, fadeOut: 0.8, ...rest }`;
      `action:'stop'|'fade'` -> `{ mode:'silence', fadeOut:
      (fadeDuration ?? 800)/1000, fadeIn: 0, boundTo:'continuous' }`;
      `action:'pause'` -> same as stop/fade (see D-09, pause is dropped).
      Preserve `assetId`, `volume`, `loop` unchanged where present, with safe
      fallback defaults if missing.
    - `migrateSoundBlockData(data: unknown): SoundBlockData` — same
      already-migrated short-circuit on presence of `mode`; otherwise map
      `action:'play'` -> `mode:'track'`, `action:'stop'` -> `mode:'silence'`;
      preserve `assetId`/`volume`/`loop`/`pitchVariation`; default
      `fadeIn`/`fadeOut` to `0`.
    - `migrateSceneRecordTimeline(timeline: TimelineStep[]): TimelineStep[]` —
      maps over steps, applying the two functions above to `music`/`sound`
      blockType steps via their `data`, leaving all other step types
      untouched.
    - `migrateSceneRecordsByStory(scenes: Record<string, Record<string,
      SceneRecord>>): Record<string, Record<string, SceneRecord>>` — applies
      `migrateSceneRecordTimeline` across every story's every scene.
    Keep all functions pure (no imports from app-store/persistence modules)
    so they can be unit tested and reused from `scene-normalizer.ts` later
    without a dependency cycle.
  </action>
  <acceptance_criteria>
    - Calling either music/sound migrator twice on its own output is a no-op
      (idempotent).
    - Legacy `action:'play'|'stop'|'fade'|'pause'` inputs all produce valid
      new-shape output with no leftover `action` field.
    - Already-migrated input (has `mode`) passes through unchanged.
  </acceptance_criteria>
</task>
