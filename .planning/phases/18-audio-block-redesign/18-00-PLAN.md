<objective>
Replace the `action`-based Music/Sound block contract with an explicit
`mode: 'track' | 'silence'` model that carries fade in/out (seconds),
scene-vs-continuous binding, and an optional auto-fade timer. Fix
`AudioPlayerService.crossFade()` so it actually fades instead of silently
ignoring its duration option. This plan only touches the data contract and
the low-level audio service â€” no runtime wiring, no migration, no UI yet
(those are 18-01 through 18-03).
</objective>

<tasks>

<task type="auto">
  <name>Redesign MusicBlockData / SoundBlockData</name>
  <files>lib/engine/types.ts</files>
  <action>
    Replace the current shape:
    ```
    export interface MusicBlockData {
      assetId: string | null;
      action: 'play' | 'stop' | 'pause' | 'fade';
      volume: number;
      loop: boolean;
      fadeDuration: number; // ms
    }
    export interface SoundBlockData {
      assetId: string | null;
      action: 'play' | 'stop';
      volume: number;
      loop: boolean;
      pitchVariation: number;
    }
    ```
    with:
    ```
    export interface MusicBlockData {
      mode: 'track' | 'silence';
      assetId: string | null;         // meaningful only when mode === 'track'
      volume: number;
      loop: boolean;
      fadeIn: number;                  // seconds
      fadeOut: number;                 // seconds
      boundTo: 'scene' | 'continuous'; // meaningful only when mode === 'track'
      autoFadeAfter?: number;          // seconds; meaningful only when mode === 'track'
    }
    export interface SoundBlockData {
      mode: 'track' | 'silence';
      assetId: string | null;          // 'track': sfx to play; 'silence': looping channel to stop
      volume: number;
      loop: boolean;
      fadeIn: number;                   // seconds
      fadeOut: number;                  // seconds
      pitchVariation: number;
      boundTo?: 'scene' | 'continuous'; // relevant only when loop === true
    }
    ```
    Do not add a `pause` mode/action anywhere in this contract (see
    18-CONTEXT.md D-09). `fadeIn`/`fadeOut`/`autoFadeAfter` are always seconds
    at this layer, matching `EffectBlockData.fadeIn`/`fadeOut` a few lines
    above in the same file â€” do not use milliseconds here.
  </action>
  <acceptance_criteria>
    - No `action` field remains on `MusicBlockData`/`SoundBlockData`.
    - `mode: 'silence'` is the only representation of "stop"; there is no code
      path anywhere in this file that treats a null/empty `assetId` alone as
      a stop signal.
    - Fade fields are documented as seconds via inline comment, consistent
      with `EffectBlockData`.
  </acceptance_criteria>
</task>
