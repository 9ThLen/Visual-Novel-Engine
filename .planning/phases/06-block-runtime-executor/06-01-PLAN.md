  <action>
    Create `lib/engine/useSceneExecutor.ts` with:

    ```typescript
    interface UseSceneExecutorOptions {
      initialVariables?: Record<string, string | number | boolean>;
    }

    interface UseSceneExecutorReturn {
      sceneState: SceneState;
      currentStepIndex: number;
      isComplete: boolean;
      isTyping: boolean;
      canAdvance: boolean;
      advance: () => void;
      selectChoice: (optionId: string) => void;
    }

    function useSceneExecutor(
      timeline: TimelineStep[],
      options?: UseSceneExecutorOptions
    ): UseSceneExecutorReturn
    ```

    Core logic:
    - Maintain `internalIndex` (auto-advances through non-yielding blocks, stops at yielding ones).
    - Expose `currentStepIndex` for UI to know which block is active.
    - `advance()` — advance past the current yielding block.
    - `selectChoice(optionId)` — record choice in variables, trigger transition.
    - `isComplete` — true when all blocks have been processed.
    - `isTyping` — true when current block is text/dialogue and user hasn't finished reading.

    Block handling:
    - **background**: set `sceneState.backgroundAssetId`, use transition+duration from data.
    - **character**: upsert character in `sceneState.characters[]` with position, sprite, visibility.
    - **text**: yielding. Set dialogue text for UI.
    - **dialogue**: yielding. Push to `dialogueHistory`, set current dialogue.
    - **choice**: yielding. Set `sceneState.currentChoices` from data.
    - **effect**: push to `activeEffects` with start/end time.
    - **music**: update `musicTrackId`, `musicPlaying`, `musicVolume`.
    - **sound**: (ephemeral — expose via callback or event).
    - **camera**: (transform only — no SceneState change, log if needed).
    - **variable**: evaluate the operation on `sceneState.variables`.
    - **transition**: yielding. Set `isTransitioning`, `transitionTarget`.
    - **interactive_object**: (overlay — no SceneState change for now).
    - Disabled blocks (`enabled: false`): skip silently.
    - Conditions: evaluate via `conditionsMet()`, skip if not met.

    The hook should NOT reference any rendering — it's a pure state machine.
    Use `useRef` for mutable state to avoid re-render overhead on every auto-advance step.
    Use `useState` only for externally-observable state (sceneState, currentStepIndex, isComplete).

    Follow the HTML prototype pattern but implement as a proper React hook with useCallback/useRef.
  </action>
