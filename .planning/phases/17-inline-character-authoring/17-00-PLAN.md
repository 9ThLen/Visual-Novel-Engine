<objective>
Create the data contract for inline character authoring: persistent character colors, user-owned named sprites, authoring defaults, explicit character actions, and sprite URI resolution.
</objective>

<tasks>

<task type="auto">
  <name>Extend character library types</name>
  <files>lib/character-types.ts</files>
  <action>
    Add persistent `color` and optional `authoring` metadata to `Character`.
    `authoring` stores future-writing defaults only: `currentSpriteId`,
    `currentPosition`, and optional `focusOnSpeak`.

    Keep `CharacterSprite.name` as the user-facing custom name. Do not replace it
    with preset emotion labels.

    Do not add persisted `hasOpenedMenu` / `firstUseMenuOpened` fields. First-use
    menu opening is an editor-session event.
  </action>
  <acceptance_criteria>
    - Existing character records remain valid with optional new fields.
    - New fields clearly distinguish authoring defaults from runtime state.
  </acceptance_criteria>
</task>

<task type="auto">
  <name>Add character library migration helper</name>
  <files>lib/character-migration.ts stores/use-app-store.ts</files>
  <action>
    Add a pure migration helper for existing character records. For each
    character:
    - preserve existing fields
    - assign `color` if absent, using the existing stable color helper or an
      equivalent deterministic palette
    - create `authoring` if absent
    - initialize `authoring.currentSpriteId` from `defaultSpriteId`
    - initialize `authoring.currentPosition` to `center`
    - initialize `focusOnSpeak` defaults without forcing focus on if product
      defaults choose it off

    Wire it into character library hydration/save paths without overwriting
    user edits. Primary entry points are `use-app-store.ts`
    `migrateFromLegacyKeys()` / hydration flow and story import paths; do not
    call this from StoryAutoSave.

    Add or update a story/character schema marker so future loads can detect
    whether character authoring metadata migration has run.
  </action>
  <acceptance_criteria>
    - Old stories load with stable colors.
    - Old characters get authoring defaults without losing sprites.
    - Migration is idempotent.
    - Migrated story data carries a detectable schema/migration marker.
  </acceptance_criteria>
</task>
