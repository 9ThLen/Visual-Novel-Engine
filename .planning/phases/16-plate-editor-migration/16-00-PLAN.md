<task type="research">
  <name>Verify installed Plate API and platform support</name>
  <action>
    Start from the actual dependency state:
    - `package.json` declares `platejs`.
    - `node_modules/platejs/package.json` currently resolves to `platejs@53.1.2`.
    - `platejs` depends on `@platejs/core@53.1.2`,
      `@platejs/slate@53.0.7`, and `@platejs/utils@53.1.2`.

    Verify the declared and installed versions match. If `package.json`,
    `pnpm-lock.yaml`, and `node_modules/platejs/package.json` disagree, stop and
    run dependency install/update intentionally before planning serializer code.

    Use Context7 first, per AGENTS.md:
    1. `resolve-library-id` for `platejs`.
    2. `query-docs` for current editor creation, plugins, custom elements,
       serialization, React support, and React Native/WebView constraints.
    If Context7 is unavailable, use local installed package types/metadata and
    official Plate documentation. Do not add packages unless the decision file
    proves they are required.

    The research must answer these questions explicitly:
    - Does `platejs@53.1.2` include the React-specific editor APIs, or is an
      additional import such as `platejs/react` / `@platejs/*` required?
    - Can web use Plate directly without iframe/WebView?
    - What is the minimum package set for a basic editor with custom elements?
    - Does Plate support custom element types for VN blocks: dialogue, choice,
      background, character, music, sound, transition, variable, effect, camera,
      interactive_object?
    - Is there a supported React Native package, or is native WebView the only
      viable runtime?
    - What node shape does Plate expect for custom block elements and void-like
      technical elements?
  </action>
  <acceptance_criteria>
    - Decision states exact package(s), installed version, and supported runtime.
    - Decision says whether existing `components/vn-plate-editor` bridge will be
      evolved or replaced, with reasons.
    - Decision states whether native uses direct RN or WebView.
    - Decision states expected Plate document node shape for VN blocks.
  </acceptance_criteria>
</task>
