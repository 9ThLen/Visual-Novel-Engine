# Phase 09.1 — Post-Migration Deferred Tech Debt

**Status:** Planned
**Created:** 2026-05-31

## Objective

Close remaining low-priority tech debt deferred from Phases 08 and 09: migrate `UserSettings` type definition to its canonical home, clean up `lib/types.ts` barrel, verify audio subsystem health, and remove dead code.

## Background Research Summary

- **`lib/types.ts` (29 LOC):** Defines `UserSettings` (6 fields) plus re-exports `PlaybackState`, `SaveSlot`, `Story`, `StoryScene`, `Choice`. JSDoc header tells new code to import from `@/lib/user-settings` — but `user-settings.ts` does NOT export `UserSettings`, only `defaultUserSettings` and `normalizeUserSettings`.
- **Audio subsystem:** All 7 audio files (`audio-types.ts`, `audio-library.ts`, `audio-library-service.ts`, `audio-manager-enhanced.ts`, `audio-player-service.ts`, `audio-trigger-scheduler.ts`, `audio-interfaces.ts`) are in active use. No dead code. Clean facade pattern.
- **`lib/i18n.ts`:** Does NOT import `expo-localization`. Language state managed via Zustand. No action needed.
- **8 consumer files** import from `@/lib/types`. 2 already have TODOs about migrating away.
